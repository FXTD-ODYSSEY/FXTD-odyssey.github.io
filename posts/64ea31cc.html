Maya MGTools&animBot 动画插件研究
发表于2020-04-06|更新于2020-08-03|神秘研究
|字数总计:2.8k|阅读时长:9分钟|阅读量:

前言

  动画工具的需求其实一直是迫切的，奈何没有那么多时间去做详细的研究。
  因此直接拿市面上成熟的插件来用是很好的方案。

  其实我好早之前就知道有 animBot 这款 Maya 做动画的神器了，忘记是在哪个 QQ群 里看到的。
  在我来到腾讯开始支持 Maya 动画组之后，感觉这种完善的动画工具需求非常大，如果有办法可以破解使用就再好不过了。

  于是在自己的期待下，我还是尽力腾出了时间来研究这些插件的破解方法。
  后来我成功破解了 animBot 插件之后，又从导师那里得知了 MGTools 插件，这个是老牌 动画 插件了。
  于是我就再接再厉，顺带把 MGTools 也破解了。

Python加密难题

  python语言编程其实有个缺点，对程序的保护不够友好。
  尽管可以通过 pyc 字节码来掩盖源码，但也起不到很好的保护作用。
  网上就可以找到一些 Decompiler 来破译源码。

  因此要保护好自己的源码，最好的方式就是通过 Cython 编译出 pyd 二进制码真正实无法破译。
  只是这样子搞就没了了 python 语言的跨平台特性，需要针对不同的平台编译对应的 二进制
  这就为啥 C++ 的软件都要有对不同平台的 release 版本，虽然执行效率高，但是使用起来就是不太方便。
  另外 Cython 编译其实也会带来一些限制，比如 __file__ 只能在函数内部使用之类的问题，的确不方便。

  虽然 python 加密比较困难，但是可以类似 js 一样通过混淆的方法将代码的可读性降低。
  这样程序执行是正常的但是破解的代码就很难理解清楚其中的逻辑和道理。

AnimBot破解尝试

  AnimBot这个软件是提供了 31 天的使用期的。
  但是试用申请需要联网申请，然后联网验证。
  否则验证机制会在点击按钮的时候立刻触发，锁定 AnimBot 的全部功能

  于是我就开始尝试破解的方法。

  从上面的截图可以看到要获取的代码完全是乱码。
  但是没有关系，我可以利用 python 的 pdb 来对源码进行 Debug
  即便没了注释 和 变量名，也还是大致可以理顺运行的逻辑的。

  不过魔高一丈，道高一尺。
  animBot 的开发者牛逼在于已经想到了我会用 Decompiler 来解译 pyc
  因此加入了一些我也不太清楚的机制，解译的 pyc 运行会报错，估计是在 字节码 层面上添加的保护机制。
  如果破解执行_api 文件夹里的代码会导致 Maya 运行报错
  我以为自己在第10层没想到作者已经千层饼了。

  并且验证机制触发的代码其实是在 _api 文件夹下的 pyd 文件里面，因此这部分的验证破解几乎是不可能的。

  不过至少我还是可以看到混淆的乱码，然后就可以通过 pdb 断点调试 pyc 结合破解的源码来分析 animBot 的内在逻辑
  而且 animbot 每个按钮都有可能触发验证机制，这就得花大量的时间来对付它的乱码(:з」∠)

利用 Qt 机制跳过锁定

  最后我经过了额一番折腾也就破解了个功能，破解的原理也很简单。
  在乱码中找到触发验证机制的函数，然后我自己写一个空函数将原有的函数覆盖掉。
  而且我这种操作也不知道会不会导致一些功能错误或者 Maya 崩溃等等问题。

  最后我左思右想决定放弃和验证机制火拼了。
  即便触发了验证机制我也可以考虑能否强制恢复 AnimBot
  毕竟 Maya 的插件都是基于 Qt 的
  验证触发之后按钮全部灰色不能点击相比只是 QWidget.setEnabled(False) 了
  这个只要重新 设置为 True 就可以继续点击了。

  下面的问题就是怎么找到那个需要设置 为 True 的组件而已。
  这个其实问题不大，我之前开发 mpdb 的时候甚至可以将这个 Maya 的 widget Tree 打印下来
  区区 AnimBot 的封锁用递归遍历组件然后调用 setEnabled(True) 恢复即可。

  恢复之后果然又可以使用 AnimBot 了，但是 AnimBot 的验证机制并不是一次性的。
  即便已经处于锁定状态的 AnimBot ， 我们次点击还是会触发验证重新给我锁定了。

  本来到这一步就基本无解了，好在我还有 Qt 的黑科技 eventFilter 来获取到 Qt 的事件序列。
  我们已经知道触发验证机制之后会弹窗警告用户的。
  而根据 Qt Event 类型这种阻挡弹窗会触发 Qt 的 QEvent.WindowBlocked 的事件
  给主窗口绑定这个事件 eventFilter 就可以获取到每次阻挡弹窗的事件了。
  下面就可以通过 QApplication.activeModalWidget() 来获取到阻挡的窗口，这里可以获取一下 弹窗的标题是否有 Animbot 判断是否要解锁。
  然后就可以关闭窗口解锁下方封锁的操作，实现 AnimBot 的破解了。

  上面的方法虽然交互体验上有点不好，但是基本实现了完美的解封，只可惜有一个功能会失效。
  那就是 AnimBot 很好用的 临时轴心点功能。
  当触发弹窗之后 临时轴心点 的选择被清空了。
  不过没有关系，可以通过上面提到的针对功能破解的方式，通过破解的源码，用空的类覆盖掉里面调用的验证函数就可以解决这里的问题。

MGTools 破解尝试

  MGTools 是老牌动画工具了，而且也相当良心了，居然提供了 20 天的免费试用。
  但是我还是要无耻地尝试破解 (:з」∠) 白嫖太香了。

  MGTools 的底层大部分都是 mel 语言写的，因此从破解的方法会和 AnimBot 不太一样。
  而且 MGTools 用到了 mll 插件，安装包里面提供的 mel 脚本也是缺失的，我当时预期可能会比 animBot 更加麻烦。
  当然 MGTools 如果只是锁定了 UI 的话，我相信也可以通过 animBot 的方式强行解锁 UI 。

  不过我研究的过程中倒是对 MGTools 是如何隐藏掉插件的脚本，并且时间校验是怎么实现的感到非常的好奇。

  通过安装包里面提供的脚本，可以看到 MGTools 的 mel 命令可以获取到当前插件的试用时间，也可以输入 license 进行注册
  并且很多代码被隐藏了。
  通过 安装包 的插件目录下可以找到 MGTools_Data.bin 文件，很有可能就是从内存里 pickle 下来的隐藏数据。
  当我删掉这个文件启动 MGTools 是会报错的。

  于是我想到无论是时间验证还是 MGTools 隐藏的脚本数据，这些东西都需要解压出来使用的，因此肯定会有文件交互。
  我现在需要一个软件来监听程序的读写，这样我就可以找到和 MGTools 相关的调用，就可以知道这些数据到底放到哪里了。

procmon 文件读写监听

  网上顺便搜了一下，windows 平台推荐微软提供的 Process Monitor ，软件小巧，功能强大。

  使用起来也非常简单，点击监听就可以监听这个系统所有程序的读写情况了。
  只是数据太大了，需要用过滤规则来过滤出我们想要的数据。

  简单过滤一下就可以发现 MGTools mll 加载的时候会读取 MGTools_Data.bin 文件
  然后进行注册表的读写操作。
  然后将隐藏的脚本解压到 Temp 路径来给 Maya 执行。
  其实也可以通过 whatIs 命令来查看 MGTools 的一些窗口脚本的代码路径，也可以发现是指向到 Temp 文件夹里面。
  不过 MGTools 解压加载到 Maya 之后就自动将这些临时的 mel 文件删除了。
  最后还是没有办法获取到隐藏的 mel 文件。

MGTools 隐藏脚本获取 | 验证日期破解

  最初我是想通过文件删除恢复的方式来恢复掉删除的 mel 文件。
  不过可能 Temp 目录的读写实在是太多，普通的文件恢复工具根本不管用。
  后来我又换了思路，通过锁定删除权限的方式锁定 Temp 目录。
  果然通过上面的方法， mel 文件无法删除就保留到了 Temp 目录里面，通过这个方法就可以获取到 MGTools 里面源码。

  破解了源码之后其实完全可以重载魔改的源码来让 MGTools 不受验证影响。
  不过我还是想破解验证机制，毕竟 MGTools 不联网，肯定有破解本地验证的方法。
  不过我总是想着是通过文件读写的方式来读取安装时间的，因此一直在文件读写的地方找了，找了好久也没有收获。
  后来我开启注册表看到上面的 twinCodes 的读取写入，由于那里没有 MG 关键字相关的东西，我还很怀疑这个注册表读写到底和 MGTools 有没有关系。
  于是我就直接搜索 twinCodes ，万万没想到 MGTools 还有 twinCodes 这个域名。
  所以可以确认这个注册表是和 MGTools 相关的。

  根据 Procmon 给的指引，可以找到注册表里面的数据。
  只要将 TrialInit 和 TrailLast 的数据同步为一样的，验证时间就是 20 天了。
  这个操作可以通过 python 的 winreg 库来操作，这样就可以实现无限试用。

总结

  这次破解历程又让我学习到了很多东西，让我对软件的验证机制有了更加深刻的了解。
  虽然我也知道破解是不好的，但是白嫖实在是太香了(:з」∠)
