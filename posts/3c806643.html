Chrome 浏览器扩展 实用推荐
发表于2020-10-17|更新于2022-07-24|效率提升指南
|字数总计:1.4k|阅读时长:4分钟|阅读量:

前言

  我原本还计划要出一篇关于 Windows 美化指南的文章。
  但是我个人是实用主义先行的，美化没啥用，还要用软件修改系统底层的东西。
  随着时间流逝，我还是奉行了简洁至上主义吧，没有把美化贯彻到底，所以 windows 美化指南也就流产了。

  这一篇文章来总结一下最近一直一段时间好用的 Chrome 插件。

  以前我常用的浏览器是 360 ，没有别的原因，最早开始用起来的。
  后面收藏夹等等东西全部都存在里面，形成了依赖，所以就一直用着。
  后面网上查了一下，发现 Chromium 浏览器的收藏都是可以互相迁移的。
  国内几乎所有的浏览器都是基于 Chromium 内核修改的（没错吐槽的就是国产之光红芯浏览器）

  后来来了腾讯之后，还是支持一下腾讯的产品，所以就转用了 QQ浏览器了。
  虽然在网上看到很多文章推荐 微软新出的 Edge 浏览器。
  不过我还是没有找到让我非转不可的理由，所以我目前还是使用着 QQ浏览器 。

Vimium

Google 商店

github 地址

  这个插件是集成大成之作，整合了多个插件的功能。
  在还没有知道这款大名鼎鼎的插件之前，我还是通过好几个不同的插件来实现一些便利的操作。
  自从发现了这款神器之后，我就少用了好多个插件。

  这款插件的功能非常多，但我个人并不常用 vim 所以并没有那么深的键盘执念。
  所以其实很多功能我都不大用得上。

  Vimium 有非常不错的搜索功能。
  通过 O 键可以快速搜索历史，就可以通过这个 vomnibar 来快速进入搜索页面

  搜索引擎配置可以在 属性的 Custom Search Engine 进行配置，不过我个人可能更习惯使用 listary

  另外 yt 快捷键可以快速复制标签页，这个我有很喜欢。
  不过我通常将这个快捷键改为 alt+shift+D ，习惯使然。

  有点不爽的就是 X 键，会和我另一个插件产生冲突，而且本事也可以通过 ctrl+W 来关闭标签，所以我就把它干掉了。

  另外 Vimium 的搜索功能支持正则表达式，有替代我另一个插件的功能。 根据 github说明
  搜索模型按 / 键启动，并且在最后加上 \r 即可启用正则表达式匹配

  只是匹配方式不太适应， Enter 键确认匹配方式之后，按 n 键来跳转匹配结果。

Tampermonkey & html5视频播放器增强脚本

Google 商店

  油猴脚本插件绝对是重量级，它相当于脚本容器，让简单的 JS 脚本也可以使用 浏览器插件的 API。
  而油猴脚本千千万万，能够让我爱不释手的只有一个 - Html5 视频播放器

github 地址

Greasy Fork 安装地址

  有了这个视频播放器增强功能，我看教程再也不需要下载了，操作习惯是按着 PotPlayer 来的。
  功能非常全面，我大多数时候只需要 视频加速 功能 和 Shift + P 在 youtube 上开小窗口模式。
  不少快捷键和 Vimium 冲突了，使用的时候需要 按 i 键开启 Vimium 的插入模式，防止快捷键冲突。

OneTab

Google 商店

  这个工具可以一键收容当前所有的窗口，然后按需要进行打开。
  用得不多，页面开多了不舍得关闭的时候还挺有用的。

Smart TOC

Google 商店

  这个小工具可以自动生成文章的目录，有时候看别人的博客没有目录浏览的时候还是非常方便。

JSONView

Google 商店

  测试接口的时候还是挺好用的，自动格式化 Json 数据，看得更舒服。

二箱

Google 商店

github 地址

  鼠标右键在 二箱 的标签下可以提取当前网页的所有图片。
  快速获取当前页面可下载的图片链接，并且可以直接拖拽到本地。
  省得我用 F12 在 Html 里面找地址了。

  另外 二箱 提供了全网快速搜图功能，非常赞~

二管家

Google 商店

github 地址

  这个和上面的插件是同一个作者开发的。
  这个管家可以对自己本地安装的插件进行分组，快速开启不同分组的插件。
  如果安装了很多插件的话，使用这个管理非常好用。

Octotree

Google 商店

github 地址

  这个插件给 Github 网站扩展了层级树结构，方便浏览项目下的文件。

总结

  其实我对浏览器插件的需求很低，日常使用上用得最多的就是 Vimium 和 H5 播放器 两个
  希望这次个人分享对大家使用浏览器有所帮助。
